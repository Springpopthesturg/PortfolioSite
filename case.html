Case Study for Texter full-stack project

Overview

texter Is a serverless native app, developed using React-Native. It provides users with a chat interface where they can send/receive text messages and images as well as share their location.


Purpose & Context

texter Was a project I built as part of my full stack web development course at CareerFoundry to demonstrate my mastery of the creation of native applications.


Objective

The aim of the project was to create an app that has: user interface customization, connects to wifi, can still run if offline, allows share users location, and has the ability to send both text and picture messages.

Approach

Client-Side

Started with having seperate views for the home page and the messaging page. Google firebase was used to store the chat so upon return the client can see old messages. All of the styling is used through react-native.

Challenges

This was both my favorite and most challenging project. I have enjoyed building the trial and error process while using a mobile emulator. The most trouble I had was writing all of my styling within javascript, but after a lot of practice and looking over my peers work it came just as natrually as css.

Duration

The development of the client-side took me twice as long as it normally took to complete a CareerFoundry achievement. It was because I needed time to wrap my head around how React-Native works.
